Behind DUFT

DUFT is developed by the University of California, San Francisco. It is built on a clean, modular architecture that separates the backend data engine from the visual frontend and configuration layer. The backend is written in Python, making it accessible to data teams already familiar with scripting and Jupyter Notebooks.

Data transformation pipelines are executed as scheduled ETL jobs or on demand, connecting to a wide range of Python-compatible databases including PostgreSQL, MySQL, SQL Server and SQLite. The frontend is built using ReactJS, delivering a fast and responsive user experience for dashboards.

Core Architecture

Built with Python and ReactJS, featuring a modular design that separates backend data processing from frontend visualization. Database agnostic and AI-ready for natural language queries.

Customization Layer

Easy configuration through JSON files - no coding required. Define dashboards, indicators, and filters with reusable components.

Deployment & Operations

Flexible deployment options with Git integration and enterprise-grade security features. Run as a central server or standalone application.
