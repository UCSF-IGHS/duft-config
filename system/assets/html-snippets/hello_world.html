Data Transformation Engine

At its core, DUFT supports powerful Python-based data transformation pipelines. Whether you're cleaning data from an OpenMRS instance or merging multiple sources into a central repository, DUFT enables you to write transformations as scripts or Jupyter Notebooks and schedule them as ETL jobs. These jobs can pull data from local or remote systems, transform it according to business rules, and upload it into a centralised repository or national data warehouse. Its compatibility with any Python-supported database makes it a highly versatile option for varied use cases—from local clinics to national-scale data systems.

Discover insights and turn them into knowledge

DUFT makes it easy to build custom dashboards tailored to programme and facility needs. Using modern, industry-grade charting libraries like ApexCharts, dashboards can include filters, search functions, data exports, and layout customisation with minimal setup. Designed to be deployed both in central servers and at health facility level, DUFT enables data use at the point of care while also supporting strategic-level visualisations for national programmes. With native support for custom layouts, users can build interfaces that match their context and use case.

Enterprise Features

Beyond its core capabilities, DUFT includes enterprise-ready features such as scheduled data transformations, user authentication and role-based access, and seamless GitHub integration for version control. Remote deployments and updates are straightforward, making it well-suited to distributed environments such as health facilities or partner organisations. DUFT can integrate AI components where needed, and despite its robustness, it remains fun to use—designed for teams who want powerful tools without the overhead. DUFT is flexible to accommodate most setups.
